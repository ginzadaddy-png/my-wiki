Note.com 게임 설계론 시리즈 (うきょう, 2026-01-11). 4년 개발에도 미완성된 4000만 엔 규모 프로젝트 실패 분석.

**실패의 구조**:
- **계약자의 심리적 인센티브**: 프로젝트 지속 = 수입 확보 → 무의식적으로 진행 방해 (나쁜 것을 나쁘다 말하지 않음, 진척 숨김, 확인 회피)
- **오너의 취약성**: 개발 비경험자 + 다른 업무로 바쁨 → "전문가니까 괜찮겠지"로 판단 정지
- **악순환**: 양측 심리가 맞물려 "알아챘을 때는 이미 늦은" 상태 진입

**방어 전략 (IT 업계 지식 응용)**:
1. **철칙 1 - 계약서를 첫 방어선으로**: 사양·전체상·담당자 바이네임·철수 기준을 사전에 명시
2. **마일스톤 가시화**: 1~2주마다 동작하는 빌드 확인. "아직 보여줄 수 없다"는 빨간 신호
3. **MVP 우선**: 핵심 재미를 가장 먼저 구현. 미완성 기능 위에 쌓지 말 것
4. **철수 기준 사전 설정**: "면접관이 말하는 기준" → "알파 테스트에서 테스터 8/10명이 '한 번 더' 응답"
