GDC 2026. 엠바크 스튜디오 프로덕션 디렉터 카이오 브라가. 7년간 개발한 [[arc-raiders|아크 레이더스]]가 기술적으로 완성됐지만 재미가 없었던 상황에서, 팀을 120명에서 25명으로 줄이고 게임을 살려낸 과정.

## 핵심 요약

1. **비전 부재가 근본 원인**: 2020년 합류 시점에 팀원마다 게임의 정의가 달랐다 — 배틀로얄, 히어로 슈터, 코옵 소울류, 레이싱… "매일 다른 게임 다섯 개를 플레이하는 상황"

2. **데이터로 현실 직면**: UXR 담당자 투입 후 내부 확신("게임이 깊고 재미있다") vs 외부 플레이어 반응이 정반대로 나타남. "게임이 가끔만 재미있었다"는 것을 인정하는 데 가장 큰 용기가 필요했음

3. **리셋: 120→25명**: 넥슨의 한 번 더 기회. 인원 감축이 오히려 강제적 집중을 만들었다 — "좋은 것을 더 좋게 다듬을 여유가 없으니, 지금 해결해야 할 문제에만 집중"

4. **기둥 + 의도 소통**: 네 가지 기둥(높은 판돈·선택과 주도권·충실도와 깊이·접근성)을 정의하고, 지시 대신 의도를 전달 → 팀원이 스스로 더 나은 해결책을 찾아냄 (자동 재장전 사례)

5. **3일 버스트 + 우선순위 바구니**: 문제 분해(1일) → 브레인스토밍(2일) → 우선순위·실행(3일). 바구니 3개(당장 해결 3개 / 여력 시 10개 / 이미 충분)로 논쟁 제거

6. **데이터가 직관을 교정**: "PVP가 싫다"는 피드백 → 데이터 분석 결과 문제는 무기 밸런스·매치메이킹. 루팅 만족도 1.4/3점 → 루팅 유용성 강화. 야간 습격 어둡게 → 실제 문제는 위험도 지각 부족

7. **세 가지 교훈**: "의도가 정렬을 만든다 / 명확함이 통제를 이긴다 / 데이터는 우리를 정직하게 만든다"

> **핵심 인사이트:** "규모가 작아서 가능했다기보다, 집중이 강제됐기 때문에 가능했다." 팀 축소는 수단이었고, 진짜 변화는 소통 방식과 의사결정 구조의 전환이었다.
