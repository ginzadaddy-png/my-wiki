발표자 5인: Jill Murray, Evan Narcisse, Kim Belair, Mike Rogers, Son M. GDC 2026.
만화 IP를 게임으로 적응시키는 전략. 여러 발표자가 각기 다른 관점으로 발표.

## 어댑테이션의 5가지 핵심 스킬 (Murray)

| 스킬 | 내용 |
|------|------|
| Analysis | IP의 세계·캐릭터·내부 논리가 어떻게 작동하는가 |
| Curiosity | 이 IP의 팬은 누구이며 무엇과 연결되는가 |
| Creativity | 어떤 시스템으로 소통할 수 있는가, 나의 목소리를 IP의 목소리로 |
| Collaboration | 승인 체계와 그 사람들의 필요/기대는 무엇인가 |
| Synthesis | 팬이 아직 원하는지 모르는 새로운 것을 IP 홀더도 흥분할 방식으로 주기 |

## 알아야 할 것 6가지

1. **프로젝트 셋업 파악**: 예산·규모·제작팀·역할 정의·승인 체계
2. **IP를 제대로 배우기**: 표면만이 아니라 깊이. 팬들이 연결되는 지점
3. **플레이어 알기**: 코어 팬 + 신규 유입 동시 만족. "클래식하면서 신선"
4. **자신만의 관점 갖기**: 개발자의 독자적 시각이 있어야 함
5. **IP 홀더와의 관계 구축**: 신뢰는 "와우"를 주려는 것보다 WIP 공유·의견 불일치 솔직히 말하기·어려운 점 인정에서 형성
6. **플레이테스트와 QA**: 타겟 오디언스(특히 IP 팬)와 반드시 테스트

## IP 홀더 신뢰 구축 (Murray)

**신뢰를 쌓는 방법**:
- WIP를 공유하고 피드백 요청
- 의견이 다를 때 정중하게 설명
- 어려운 점을 인정하고 도움 요청
- 한계를 명확히 설정하고 지키기

**신뢰를 못 쌓는 방법**:
- 클라이언트를 "와우"시키려고만 함
- 모든 요청에 반사적으로 동의
- 문제가 생겼을 때 숨기기

## 슈퍼히어로 IP 협업 (Narcisse)

- 게임이 원작의 공백을 채울 기회 (Miles Morales 사이드퀘스트: 최소 1명의 마블 캐릭터 등장 조건)
- 팀워크가 핵심. 퍼블리셔는 방해물이 아니라 파트너
- "규칙 중 어떤 것을 함께 깰 수 있는지 발견하라"

## 살아있는 창작자와 협업 시 주의

- 진화하는 IP → 규칙과 세계도 변함
- 여러 이해관계자의 균형 조율
- 뉘앙스를 받아들이기

## 인디 게임과 멀티미디어 (Son M.)

- 세계관 구축의 중요성과 공간을 게임에서 충분히 확보
- 캐릭터 아크: 게임에서 다 탐색하지 않아도 되는 요소를 만화로
- 사랑받는 서브 캐릭터를 추가 콘텐츠로 확장
- 두 게임 사이의 전환을 만화로 브릿지

> **핵심 인사이트:** 어댑테이션의 결과물은 "코어 팬의 기대를 충족하면서도 독자적으로 만족스러운 게임"이어야 한다. IP 홀더와의 관계는 쇼맨십이 아닌 투명성으로 쌓는다.
