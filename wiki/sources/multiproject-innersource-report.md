캡콤·프롬소프트웨어·너티독·인섬니악·엠바크 등 주요 스튜디오의 멀티 프로젝트 조직 구조를 비교 분석하고, 이너소스(InnerSource) 모델을 대안으로 제시한 보고서.

## 스튜디오별 모델 요약

### 캡콤 — 중앙집중형 + 급진적 투명성

- RE ENGINE 팀: AI·렌더링·물리·네트워크 등 3~4명 단위 세분화
- 1,300명+ 참여 팀즈 채널 개방 운영, 엔진 소스코드 전 개발자 열람 가능
- 하루 평균 20건+ 직접 문의 지연 없이 처리 → 병목 원천 차단
- **핵심 철학**: 엔진은 강제 규율이 아니라 게임 팀을 위한 "협업적 서비스"
- 전담 매니지먼트 유닛(행정 조율·설문·프로젝트 조직화)까지 배정

### 프롬소프트웨어 — 유동적 리소스 풀 + 기술 영속성

- 개발 인력 300명, 피크 시 200~230명 집중 투입 (엘든링+AC6 교차 운용)
- 자체 엔진 'Dantelion' 고수 → 프로젝트 간 이동 시 온보딩 비용 사실상 제로
- IP가 달라도 애니메이션·골격·몬스터 AI·레벨 패턴 광범위 재사용
- 기술적 재활용의 흔적을 압도적 아트 디렉션으로 은폐

### 너티독 — 수평 조직의 붕괴 → 프로덕션 관리 체계 도입

- 역사적으로 프로듀서·인사관리자 직군 의도적 기피 ("창의성 억압의 관료적 적폐"로 간주)
- 결과: 언차티드3 마감 → 라스트오브어스 팀 인력 지속 탈취, "승인 교착 상태(Sign-off Gridlock)"
- TLOU2에서 하루 12시간+ 강제 야근 일상화
- 2021년부터 전담 프로듀서 대거 고용 + 엔지니어링 관리자 트랙 공식 도입
- 현재: 하나의 메이저 프로젝트가 리소스 대부분 점유, 나머지는 프리프로덕션 대기

### 인섬니악 — 창의성 분산 + 기술 중앙화

- ~450명 규모, 병렬 다중 프로젝트 성공
- CEO의 원칙: "리더의 역할은 창의적 결정에 참견하는 것이 아니라 개발자들의 방해물을 치워주는 것"
- 핵심 기술 부서(Core Technology Department)가 엔진·툴킷·테크니컬 아트 파이프라인 전담 지원
- 각 프로젝트 디렉터는 창의적 영역에서 완전한 독립성 보장

### 엠바크 스튜디오 — 오픈소스 기반 비선형 스케일링

- ~350명으로 AAA 멀티 프로젝트, 기존 AAA 예산의 1/4 수준
- 핵심 언어: C++ → Rust (메모리 안전성, 레이스 컨디션 원천 차단)
- "오픈소스 우선" 철학: 자체 코어 기술 GitHub 공개 → 외부 커뮤니티 기여 유도
- "AAA를 만들기 위해 반드시 수천 명의 인력이 필요하지 않다"

## 실패 사례: EA 프로스트바이트

- FPS 특화 엔진을 RPG(바이오웨어 등) 게임 팀에 강제 이식 → 3인칭 카메라·인벤토리·파티 로직 없음
- 엔진 팀을 비용 부서(Cost Center)로 취급 → 즉각 지원 불가, 스튜디오 "지원 악몽"
- 인력 빼가기: 엔진 숙달 개발자를 스튜디오 의사 무관 차출 → 기술 부채 누적·번아웃·이탈
- **교훈**: 강압적 중앙집중화는 기술·조직 양면에서 참사를 초래

## 성공 사례: 소니 ICE 팀 — 하이브리드 접근

- 퍼스트 파티 스튜디오에 단일 엔진 사용 **강제하지 않음**
- 렌더링·그래픽·디버깅·프로파일링 도구를 개발해 선택적 제공
- 게릴라(데시마), 산타모니카(독자 엔진) 등 각 스튜디오 기술 자율성 완전 보장
- **교훈**: 중앙 기술 연구의 경제적 이점 + 창작자 자율성을 동시에 달성
