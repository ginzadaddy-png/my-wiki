문제 정의 — 비-라이브 서비스의 구조적 챌린지

콘솔·PC 본진 게임의 표준 개발 사이클은 5~7년에 이른다. 매출은 출시 시점에 폭발적으로 집중되고, 그 직후~다음 작품 사이의 다년간 공백 동안 인력 운영 변동성이 급격히 커진다. 라이브 서비스가 매월 매출을 만들어 인력을 평탄하게 유지할 수 있는 것과 정반대 구조다.

  • 출시 사이클의 캐시플로 변동성 — 한 작품 매출이 다음 작품 개발비를 지탱해야 함
  • 인력 잉여·부족의 주기적 발생 — 프로젝트 종료 시 일부 직군에 잉여, 신규 프리프로덕션 진입 시 부족
  • 축적된 IP 지식의 휘발 — 인력 이동·해고가 잦으면 시리즈 IP의 *암묵지*가 사라짐
  • 도구·엔진 누적 효과 상실 — 자체 엔진은 그 위에서 일하는 인력이 지속될 때만 진화

Capcom — 인재 파이프라인 4축

①

채용

연 100명+ 신규 개발자, 2013년부터 명시 정책. 채용 시점부터 실제 프로젝트 멤버화하여 5년 내 핵심 인력 성장.

②

세대 혼합

20대~50대 전 세대가 한 팀. "technology transfer"라는 IR 용어로 명시. 신입이 시니어 옆에서 의사결정 흡수.

③

도구·환경

RE ENGINE 자체 엔진 + 이너소스 1,300명 채널 + R&D Building #3 (2027 완공) + Gen AI 정식 등재.

④

보상·환경

이익 연동 보너스, 평균 연봉 ¥10.1M → ¥10.5M. 자본 배분 3원칙 중 직원 보상이 주주환원·재투자와 동등.

Case 2 — Insomniac (창의/기술 분리)

~450명 규모로 Marvel's Spider-Man 시리즈, Ratchet & Clank, Wolverine을 동시에 진행한다. 다수의 AAA를 병렬로 굴리는 보기 드문 사례. 핵심은 두 가지 축의 명시적 분리다.

  • Core Technology Department — 엔진·툴킷·테크니컬 아트 파이프라인 전담. 모든 프로젝트가 공유하는 기술 인프라.
  • 프로젝트별 디렉터 독립성 — 창의적 결정은 디렉터·팀 자율. 경영진이 콘텐츠에 개입하지 않음.
  • CEO 원칙 — "리더의 역할은 창의적 결정에 참견하는 것이 아니라 개발자들의 방해물을 치워주는 것."
  • 결과 — 신작 출시 사이 공백이 거의 없음. 한 작품 마감기에도 다른 작품의 프리프로덕션이 진행 중.

Case 3 — FromSoftware (피크 엇갈리기)

유동적 인력 배치
  • ~300명 개발 풀, 고정 팀 아님
  • 프로젝트 단계·상황에 따라 인력 흐름 재배치
  • 한 프로젝트 피크 최대 200~230명
  • AC5(2012) ~80명 → AC6(2023) ~200명, 10년 자연 성장
시리즈 IP가 경제적 토대
  • Elden Ring 30M+, Dark Souls 시리즈 33M+
  • Elden Ring 출시 후 Dark Souls III 6년차 매출 상승 — cross-sell 정량 증거
  • Kadokawa Game 부문 매출 95.6B JPY (60% Elden Ring 카탈로그 추정)
  • Sony 지분 인수 시도 IP valuation ~$2B (외부 추정)

출처: fromsoftware-300-devs · Kadokawa FY24/3 IR · catalog-economics-cross-company-2026

Case 4 — Embark Studios (규모 통제)

DICE 출신 패트릭 수더룬드가 2018년 설립. 넥슨 자회사. 약 360명 규모로 Arc Raiders (2026년 3월 기준 1,400만 장)·The Finals 두 게임을 운영하고 4개 게임 포트폴리오를 *인원 폭발 없이* 달성하는 것이 명시 목표다.

  • "스파이럴 아웃 안 함" 원칙 — 인원 늘리는 대신 *파이프라인 재설계*에 투자. 포토그래메트리·절차적 지형 생성·구식 워크플로우 교체.
  • Rust 채택 — C++ 대신 Rust로 코어 기술 작성. 메모리 안전성·레이스 컨디션 원천 차단으로 디버깅 시간 감축.
  • 오픈소스 우선 — 자체 코어 기술 GitHub 공개 → 외부 커뮤니티 기여 유도. "AAA를 만들기 위해 반드시 수천 명의 인력이 필요하지 않다."
  • 아크 레이더스 리셋 (2023) — 개발 중 팀을 120명에서 25명으로 축소. 기술적 완성도가 아닌 *재미*를 검증한 후 재확장.

Case 5 — Larian Studios (EA 기반 점진 확장)

벨기에 겐트 본사의 RPG 전문 스튜디오. 글로벌 22시간 개발 체제로 운영. 인력은 단번에 폭발하지 않고 작품 매출이 검증된 후 다음 작품을 위해 단계적으로 확장하는 사이클을 반복한다.

  • 점진 매출 곡선 — DOS1 300만 → DOS2 800만 → BG3 15M+ (외부 추정). 각 작품 매출이 다음 작품 개발비를 지탱.
  • 3년 EA 검증 도구 — BG3는 2020-10~2023-08 동안 액트 1만 공개. 텔레메트리 기반 액트 1 반복 재편집. EA 250만 카피 = 시장 검증 + 캐시플로 + 인력 안정성.
  • 모델 옹호 — Vincke: "EA를 성공적으로 졸업한 게임은 모델을 남용하지 않는다. BG3가 3년 동안 EA였다는 걸 알지?"
  • 완결 IP 선언 — BG3 후 D&D 라이선스 반납, 오리지널 IP로 전환. 시퀄·확장팩 없음 명시 → 다음 IP로 인력 이동.

Case 6 — Hazelight (단일 비전 + 고정 규모)

스웨덴 스톡홀름. Josef Fares 설립·디렉터. EA 퍼블리싱. 50~60명 규모를 8년 동안 일관되게 유지하면서 A Way Out (2018) → It Takes Two (2021, GOTY 다수) → Split Fiction (2025) 순차 출시.

  • 단일 비전 — "모든 게임은 2인 협동 전용." 솔로 플레이 없음. 모든 의사결정의 필터.
  • 고정 규모의 이점 — 한 작품 종료 후 인력 잉여 없이 다음 작품으로 즉시 전환. 권고사직·정리해고가 구조적으로 발생할 여지가 없음.
  • 플레이테스트 — 짝 단위 검증 — Fares: "파트너를 데려와 테스트한다. 끊임없이 대화해야 하니 협력 수준 자체를 시험하게 된다."
  • Friend Pass — 구매자 1명이면 친구 1명 무료 플레이. 진입 장벽 최소화로 누적 매출 곡선 유지.

Case 7 — 소규모 응용 (Supergiant·Asobo·Sandfall)

~25명

Supergiant

이직률 0% · 노 크런치 + 스코프 제한. 배스천→트랜지스터→파이어→하데스→하데스 2 시리즈화. 그렉 카사빈: 하데스 EA를 "TV 파일럿 에피소드"로 프레임화 → 매월 캐릭터·관계 반응 학습.

~20명 아티스트

Asobo Studio

A Plague Tale Requiem 기준. 인하우스 엔진 + Quixel Megascans 보조 + Midpoly 메시 전략. "Focus on what matters"가 에셋·레벨·엔진 전 결정의 필터.

~40명

Sandfall Interactive

클레르 옵스퀴르: 33원정대 (2025 GOTY 최다 수상). 프로그래머 4명, 블루프린트 95% 활용. 디자이너 도구화 구조로 소규모 인력 한계 돌파.

②

멀티 프로젝트

인력 풀의 보험 — 피크 타임 엇갈리기로 잉여 완화

③

자체 엔진·이너소스

스파이럴 아웃 방지 — 인원 대신 파이프라인에 투자

5축은 *결합*될 때 작동한다. 단일 축만으로는 지속 불가능.

축① 카탈로그 매출 — 캐시플로 안정성

캐시플로 변동성을 완화하는 가장 강력한 도구. 한 작품의 출시 후 5~9년 누적 매출이 차기작 개발비와 인력비를 지탱한다. 사례별 모델은 다르지만 본질은 동일하다.

  • Capcom — 재발매·시리즈 cross-sell — 카탈로그 비중 84%. RE7(2017)이 9년차에 260만 장. 시리즈 통합 마케팅으로 신작 출시 시 6~7개 카탈로그 동시 점프.
  • Take-Two — RCS 서비스형 — FY24 net bookings의 78%가 recurrent (GTA Online 중심). GTA V 13년차에도 분기당 5M units.
  • FromSoftware — 신작 cross-sell — Elden Ring 출시 후 Dark Souls III 6년차 매출이 *전년 대비 상승*. Kadokawa Game 부문 매출 ~60% 카탈로그 추정.
  • Nintendo — 가격유지형 — Mario Kart 8 DX 8년 우상향, 정가 $59.99 유지. 세일 최대 33%만.

축② 멀티 프로젝트 — 인력 풀의 보험

단일 작품에 모든 인력을 묶지 않음으로써 인력 풀의 보험을 만든다. 다만 *진정한 동시 풀 프로덕션*은 산업 최고 수준 스튜디오도 달성하지 못한 어려운 목표다. 현실적 모델은 단계적·유동적 배치다.

  • FromSoft 모델 — 피크 엇갈리기 — 300명 풀을 두 프로젝트가 시간차로 흐름. 한쪽 마감기에 다른 쪽 프리프로덕션 진입.
  • Insomniac 모델 — 병렬 + 기술 공통 — Core Technology Department가 공유 인프라 담당. 프로젝트 디렉터는 콘텐츠 독립성 유지.
  • Embark 모델 — 규모 통제 멀티 — 360명으로 2~4개 게임 목표. 인원 폭발 대신 파이프라인 효율화.
  • 너티독 반례 — 2013년 TLOU + UC3 동시 시도 → UC3 마감에 TLOU 자원 잠식. 2021년 시점 공동 대표 인터뷰: "현재도 아슬아슬하게 미달."

축③ 자체 엔진·이너소스 — 이동 비용 절감

프로젝트 간 인력 이동의 비용을 구조적으로 절감한다. 모든 개발자가 같은 엔진·툴킷·문서를 공유하면 새 프로젝트로 옮길 때 학습 곡선이 거의 없다. 이너소스는 그 위에 *기술 공유의 투명성*을 더한다.

  • Capcom RE ENGINE — 급진적 투명성 — 엔진 소스코드 전 개발자 열람 + 1,300명+ 팀즈 채널 완전 개방. 하루 평균 20건+ 직접 문의를 지연 없이 처리.
  • Insomniac Core Technology Department — 엔진·툴킷·테크니컬 아트 파이프라인을 별도 부서가 전담. 프로젝트는 *콘텐츠*에만 집중.
  • Asobo 인하우스 엔진 — 2002년 창립 이래 자체 엔진 유지. 20명 아티스트로 AAA 비주얼 가능한 기반.
  • Embark Rust 채택 — 메모리 안전성으로 디버깅 부담 감축 + 코어 기술 GitHub 공개로 외부 기여 흡수.

축④ 세대 혼합 — 기술 전수와 학습 곡선

매년 일정 규모를 신규 영입하고 세대 혼합 팀에서 기술 전수를 통해 5~10년 후 핵심 인력으로 키우는 모델. 단발 채용·외주 의존이 아니라 시간이 갈수록 조직 역량이 누적된다.

  • Capcom — 명시 정책 — 연 100명+ 채용 + 20대~50대 전 세대 혼합. IR 자료에 "technology transfer" 키워드 정식 등재.
  • Supergiant — 0% 이직률 — 업계 평균 30~40% 대비. 스코프 제한 + 노 크런치 + 심리적 안전감의 결과. 장르를 작품마다 바꾸면서도 팀 구성 유지.
  • FromSoft — 디렉터 중심 지속성 — 미야자키 직접 개입 모델로 핵심 인력 장기 유지. 시리즈 IP의 암묵지가 조직에 누적.
  • Hazelight — 작품 간 학습 전이 — A Way Out의 동적 split-screen 학습 → It Takes Two의 정적 split 결정. 같은 팀의 작품 간 학습.

축⑤ 규모 통제·도구화 — 스파이럴 아웃 방지

인원이 늘어나면 커뮤니케이션 비용이 제곱으로 늘어나고 의사결정 속도가 느려진다. 본진 콘솔·PC 게임에서 인원 대신 *파이프라인·도구·자동화*에 투자하는 명시 전략의 사례.

  • Embark Söderlund — "600~800명이 될 이유가 없다. 우리가 스튜디오를 시작한 이유를 잊지 않겠다." 360명으로 4 게임 목표.
  • Asobo "Focus on what matters" — Midpoly 메시 전략, Quixel Megascans 보조, Displacement→Blend Map 재활용. 20명 아티스트로 AAA.
  • Sandfall 디자이너 도구화 — 프로그래머 4명, Blueprint 95% 활용. 디자이너가 직접 게임플레이를 빌드하는 구조.
  • Capcom Gen AI 정식 등재 — 리서치·드래프트·매뉴얼·에러 체크·회의록을 AI가 처리. 시니어의 멘토링 시간을 창작으로 재배치.

반례① EA Frostbite — 강제 중앙화의 함정

이너소스의 *자발적 기여* 모델과 정반대 패턴. EA는 Frostbite 엔진을 사내 표준으로 *강제*했고 엔진 팀을 비용 부서(Cost Center)로 취급했다.

  • FPS 엔진 → RPG 팀에 강제 이식 — 바이오웨어 등 RPG 팀이 3인칭 카메라·인벤토리·파티 로직을 처음부터 만들어야 했음.
  • 엔진 팀을 Cost Center로 분류 — 외부 팀 요청에 즉각 지원 불가. 우선순위는 본진 FPS 프로젝트.
  • 인력 차출 강요 — 엔진 숙달 개발자를 스튜디오 동의 없이 빼감 → 기술 부채·번아웃·이탈 누적.
  • 교훈 — 강압적 중앙집중화는 기술·조직 양면에서 실패한다. Capcom RE ENGINE처럼 *지원 조직이 아닌 협업적 서비스*로 포지셔닝되어야 한다.

반례② Double Fine — 과잉 다각화

멀티 프로젝트는 *기술 인프라 공통화*와 *피크 타임 엇갈리기*가 전제될 때 작동한다. Double Fine의 2014년경 6+ 동시 프로젝트 시도는 그 조건이 없을 때 발생하는 실패 패턴을 보여준다.

  • 6+ 동시 프로젝트 — Massive Chalice, Spacebase DF-9, Hack 'n' Slash, Broken Age 등.
  • 전문가 시간 분산 — AI 프로그래머·FX 아티스트 등 한 명이 여러 팀의 요구를 동시 처리 → 작업 지연.
  • 결과 — 생산성 전반 하락, Spacebase DF-9 등 미완성 출시, 복수 게임 지연·취소.
  • 교훈 — 멀티 프로젝트의 임계점이 있다. FromSoft 2개·Insomniac 3~4개는 작동하지만 6+는 *집중력 분산*이 임계를 넘는다.

반례③ 너티독 — 자원 잠식의 구조적 위험

멀티 프로젝트의 또 다른 실패 모드: *우선순위가 높은 프로젝트가 다른 프로젝트의 자원을 빨아들이는* 현상. 너티독은 2013년경 The Last of Us와 Uncharted 3의 동시 풀 프로덕션을 시도했다.

  • UC3 마감 → TLOU 팀 잠식 — UC3가 임박하자 핵심 인력이 TLOU에서 UC3로 이동. TLOU 일정 지연.
  • 2021년 시점 — "아슬아슬하게 미달" — 공동 대표 에반 웰스: "두 프로젝트가 각각 수백 명을 가진 상황은 없다."
  • 현재 모델 — 메인 프로젝트 1개 + 프리프로덕션 여러 개. 진정한 동시 풀 프로덕션은 *포기*.
  • 교훈 — 산업 최고 수준 AAA 스튜디오도 동시 풀 프로덕션은 어렵다. 멀티는 *단계 시차*가 핵심.

글로벌 적용 함의 — 사전 조건

본진 콘솔·PC 게임을 지속적으로 만들어내는 스튜디오들에서 일관되게 관찰되는 사전 조건. 이는 특정 시장의 노동 관행 문제가 아니라 *비즈니스 모델 설계*의 문제다.

  • 1) 시리즈 IP 자산 또는 일관된 비전 — Capcom·FromSoft 같은 시리즈 IP 자산 또는 Hazelight·Supergiant 같은 *일관된 단일 비전*이 작품 간 학습 전이를 가능하게 한다.
  • 2) 자체 엔진 또는 도구 표준 — RE ENGINE·Asobo 인하우스 엔진·Sandfall Blueprint 95% — 어떤 형태로든 *프로젝트 간 도구 공통화*가 인력 이동 비용을 절감한다.
  • 3) 멀티 프로젝트 또는 점진 확장 — 단일 작품 ALL-IN을 피해 인력 풀의 변동성을 완화. FromSoft·Insomniac·Larian의 공통점.
  • 4) 규모 통제와 도구 투자의 균형 — 인원 폭발 대신 파이프라인·자동화·세대 혼합으로 효율을 끌어올리는 명시 전략.
