짧은 주기로 프로토타입을 제작·검증하는 개발 방법론. [[team-asobi|팀 아소비]]와 [[larian-studios|라리안 스튜디오]] 모두 핵심 개발 철학으로 채택.

**팀 아소비 방식 (아스트로봇)**
- 2주 단위: 기획 → 프로토타입(1인이 3D·애니·사운드·햅틱·코드 전담) → 전 팀원 금요일 리뷰
- 아이디어의 10%만 프로토타입으로 진행. 전체 개발 기간의 1/3이 프로토타이핑
- 검증 통과 후 아트팀 투입 → 핵심 재미가 보장된 상태에서 외형 집중
- 6년 전 폐기된 프로토타입이 후에 활용되는 경우도 있음

**라리안 방식 (발더스 게이트3)**
- 얼리 액세스를 사실상 대규모 프로토타입으로 활용
- 액트1 공개 → 피드백 반영 → 액트2·3 제작 → 액트1 재제작 → 정식 출시

**서커펀치 방식 (고스트 오브 쓰시마·요테이)**
- 반복 프로토타이핑으로 [[vision-statement|비전 선언문]]의 '반짝임' 탐색
- GotS 패링 시스템: 6개월 기반 작업 + 4개월 실험 = 10개월 만에 발견
- "프로토타입의 목표는 기능 추가가 아니라, 처음에 설정한 질문에 답하는 것"
- 목표는 일관성이 아닌 마법(반짝임). 일관성은 발견 후에 만들 수 있다

> **핵심 인사이트:** 프로토타이핑의 목적은 '완성도'가 아닌 '반짝임 검증'이다. 검증 가능한 최소 단위로 빠르게 만들고, 틀리면 빠르게 버리는 것이 핵심.
