사운드를 *시각 위에 얹는 장식(cosmetic)*이 아니라, 게임 상태 변화를 직접 전달해 플레이어의 조작을 구동하는 **1차 피드백 채널(Primary Feedback Channel)**로 설계하는 원칙. 청각 인지-운동 반응(~80–100ms)이 시각 정보 해석(~200–250ms)보다 빨라, 정교한 오디오 정보 설계는 곧 전술 우위로 직결된다.

## 세 가지 설계 원칙

| 원칙 | 내용 |
|------|------|
| 동적 정보 전달성 | 출력 주파수·음압을 게임 상태 변수(체간·가드 품질·위협도)와 고정밀 동기 → 보지 않고도 피드백 폐루프 완성 |
| 주파수 위계 | 고주파 = 완벽한 성공, 저주파 탁한 노이즈 = 미흡한 실패 → 질감의 쾌/불쾌만으로 가이드 |
| 소닉 검증 (Sonic Validation) | 타격 후 배음·긴 잔향(Ringout)으로 조작 정당성·카타르시스 부여 → 입력 본능 강화 |

## 사례

### Sekiro — 체간 시스템의 청각 피드백
[[sekiro|세키로]]는 완벽 패링(맑은 고주파 "CLANG")과 일반 가드(둔탁한 저주파 "ding")를 주파수·잔향으로 극명히 분화해, 플레이어가 *UI 체간 바를 보지 않고 소리만으로 공방 주도권 전환을 인지*하게 만든다. 위험(危) 경보음·인살 파열음까지 합쳐 전투가 "칼날 사이의 리듬 게임"이 된다. ([[sword-sfx-satisfaction-2020]])

### Overwatch — "Play by Sound"
[[gdc16-overwatch-play-by-sound|오버워치]]는 *"모니터를 꺼도 플레이 가능"*을 목표로, 정적 거리 감쇠 대신 **최대 위협 시스템**으로 매 프레임 위협도를 계산해 믹스를 재조정한다. 적대 영웅은 명료도 최대, 아군은 덕킹, 발소리는 영웅별 시그니처로 필터링해 화면 밖 정보를 청각으로 보정한다.

### 확장 — 접근성
같은 원리가 접근성으로 확장된다. TLOU2는 트래버설 비프(트랜지언트 유무로 안전 위치 신호)와 3D 듣기 소나로, 전맹 플레이어가 화면 UI 없이 조작을 완성하게 한다. ([[sound-design]] 참조)

> 💡 **핵심 인사이트:** 사운드가 게임 변수를 *은폐*하지 않고 *드러낼* 때, 플레이어는 화면을 보지 않고도 입력-피드백 폐루프를 닫는다. [[combat-design|전투 디자인]]의 가독성, [[player-guidance-design|플레이어 유도]], 접근성이 모두 이 한 원칙 — "소리를 1차 입력 장치로 대우" — 에서 갈라져 나온다.
